系统分为两级用户：普通用户和管理员用户。

普通用户使用的网络交易系统包括以下的12个功能模块：
（1）登录模块：使用者需要填上所需要的内容，等待程序检验填入的内容是否正确。如果填入的信息有误，会有提示的信息出现的。
（2）注册模块：用户填入个人信息时，若填入信息格式不符合要求，会有提示出现，如“用户名不能为空”，“邮箱格式错误”等。若填入的用户名或者邮箱已经被注册过，也会出现错误提示信息。
（3）忘记密码：以邮箱验证的方式重置密码。
（4）查看并修改个人信息：只有在登录状态下才可以修改个人信息。
（5）商品分类显示：通过点击不同的类别，分页显示同类商品。
（6）关键字搜索商品：可以通过关键字搜索对应的商品。
（7）查看商品详情：主要是显示某个商品信息的页面，如果有和该商品类似的商品，也会在页面的下方显示出来，引起用户的兴趣。
（8）购物车模块：对于某一件商品，可以点击其下方的链接，将它加入购物车中，并进行结算，并且可以随时删除购物中里面的某些商品，并可以直接清空购物车。
（9）收藏夹：将商品收入收藏夹，也可以移除收藏夹中商品，若该商品已经被加入购物车，则会提示收藏夹中已有该商品。
（10）订单模块：在购物车中提交订单后，填写自己的收货信息，然后选择某银行对订单进行支付，本系统采用第三方的支付软件易宝，进行网银支付。
（11）留言板模块：用户可以通过点击“意见建议”链接写出自己的意见和建议，并进行提交。
（12）在线客服：利用TQ的在线客服系统。

管理员登录的系统包括以下五个模块：
（1）用户管理：可以分页浏览所有用户的信息，也可以删除用户。
（2）商品类别管理：商品的类别管理分为两个，分别是对一级分类的管理（增删改查）和对二级分类的管理（增删改查），一级分类和二级分类是一对多的关系。
（3）商品管理：对商品增删改查的操作。
（4）订单管理：可以分页查看所有用户的下单情况以及的订单信息，查看订单的详情，也可以对订单进行操作，更改订单状态，并和前台购物系统进行交互，实时地更新用户的订单状态。
（5）留言管理：对网站的建议内容进行增删改查操作。

使用的技术：struts2、spring、hibernate三大框架整合技术。
网站前端的页面主要使用jsp、html+css、javascript、ajax，部分页面使用easyui进行简单的布局。
